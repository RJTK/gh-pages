This is a detailed example tying together quadratic optimization, eigenvalues, and the margin-of-safety principle applied to the Markowitz portfolio problem.

This post was originally going to be an example section for my post about quadratic forms . However, it was getting way too long, so I just broke it out into it’s own page. Essentially, there are at least two more nice quadratic-form problems that I find interesting, and there is a modest relationship between them. The first is a version of the famous Markowitz portfolio problem . This problem can then be generalized into a robust Markowitz portfolio, from which one encounters the second problem: the Trust Region Subproblem (TRSP). This TRSP will be optimized by combining bisection search with an eigenvalue decomposition .

Duality for Linear Equality Constrained Convex Quadratic Programming

A name can be given to the problem of minimizing a convex quadratic form subject to a linear equality constraint – it is a linearly constrained convex quadratic program:

\begin{equation} \begin{aligned} \underset{x \in \R^n}{\text{minimize}}&\quad \frac{1}{2}x^{\mathsf{T}} Q x + q^{\mathsf{T}} x + \frac{1}{2}\kappa\\ \text{subject to}&\quad Ax = b. \end{aligned}\notag \end{equation}

I wrote previously about how one can use the pseudo-inverse \(A^\dagger\) to eliminate the equality constraint and turn this problem into a more vanilla unconstrained quadratic form. There is another insightful approach that also eliminates the constraint, which is the approach by Lagrangian Duality. Define a function

\begin{equation} \begin{aligned} \mathcal{L}(x, \nu) &= \frac{1}{2}x^{\mathsf{T}} Q x + q^{\mathsf{T}} x + \nu^{\mathsf{T}} (Ax - b) + \frac{1}{2}\kappa\\ &= \frac{1}{2}x^{\mathsf{T}} Q x + (q + A^{\mathsf{T}} \nu)^{\mathsf{T}} x - b^{\mathsf{T}} \nu + \frac{1}{2}\kappa. \end{aligned}\notag \end{equation}

whenever \(x\) is feasible (i.e., \(Ax = b\)) the value of \(\mathcal{L}(x, \nu)\) is equal to the value of the original objective f(x)$, regardless of the value of \(\nu\). On the other hand, if \(Ax = b\) is not feasible, then \(\nu\) can be chosen to make the value of \(\mathcal{L}\) as large as possible (since we’re minimizing, think “large = bad”). We can view this as a game wherein we are the $x$-player attempting to minimizing \(\mathcal{L}\) against an adversarial $ν$-player attempting to maximize \(\mathcal{L}\). It is a useful property that the function \(\rho(x) = \underset{\nu \in \R^m}{\text{sup}}\ \mathcal{L}(x, \nu)\) is just another way to encode the quadratic program since \[\rho(x) = \frac{1}{2}x^{\mathsf{T}} Q x + q^{\mathsf{T}} x + \frac{1}{2}\kappa\] if \(Ax = b\) and \(\rho(x) = \infty\) otherwise.

Now, what if we swap the order of maximization and minimization? If our adversary \(\lambda\) has to act first, and we can choose \(x\) based on their choice, then we must necessarily obtain a smaller value for \(\mathcal{L}\):

\begin{equation} \underset{x \in \R^n}{\text{inf}}\ \underset{\nu \in \R^m}{\text{sup}}\ \mathcal{L}(x, \nu) \le \underset{\nu \in \R^m}{\text{sup}} \underset{x \in \R^n}{\text{inf}}\ \mathcal{L}(x, \nu), \notag \end{equation}

this is called the minimax inequality.

What this results in is another linear system of equation:

\begin{equation} \begin{aligned} Qx + A^{\mathsf{T}}\nu &= -q\\ Ax &= b, \end{aligned}\notag \end{equation}

which we can write in matrix notation as:

\begin{equation} \begin{bmatrix}Q & A^{\mathsf{T}}\\ A & 0\end{bmatrix}\begin{bmatrix}x \\ \nu \end{bmatrix} = \begin{bmatrix} -q \\ b \end{bmatrix}.\notag \end{equation}

Thus, we can find a complete primal-dual solution to the original quadratic programming problem by solving this augmented system of equations, which is called a KKT system . It is worthwhile pointing out that this system is symmetric, but not positive-definite.

The analytic expressions for the solutions \(x, \nu\) can be cumbersome, but just like in the unconstrained case where \(x^\star\) is a linear function of \(q\), the constrained solution is a linear function of \((q, b)\). The details are in hidden in the dropdown here, but it can generically be written simply as \(x^\star = Rq + Cb\), and the value of the problem is a quadratic form in these variables.

The first linear system of equations can be solved for \(x\) to show that \(x\) is a linear function of \(q\) and the dual vector \(\nu\):

\begin{equation} \begin{aligned} Qx + A^{\mathsf{T}} \nu &= -q\\ \implies x^\star &= -Q^{-1}[A^{\mathsf{T}} \nu + q]. \end{aligned}\notag \end{equation}

We then need to solve for the dual vector \(\nu \in \R^m\) itself, which can be done by finding the \(\nu\) which makes the constraints satisfied:

\begin{equation} \begin{aligned} Ax &= b\\ \iff -AQ^{-1} [A^{\mathsf{T}} \nu + q] &= b\\ \iff -AQ^{-1} A^{\mathsf{T}} \nu &= b + AQ^{-1} q\\ \iff \nu^\star &= -\bigl(AQ^{-1} A^{\mathsf{T}} \bigr)^{-1} [b + AQ^{-1} q]. \end{aligned}\notag \end{equation}

We then substitute this back into the expression for \(x^\star\) to get

\begin{equation} \begin{aligned} x^\star &= -Q^{-1} [A^{\mathsf{T}} \nu^\star + q]\\ &= -Q^{-1} q + Q^{-1} A^{\mathsf{T}} \bigl(AQ^{-1} A^{\mathsf{T}} \bigr)^{-1} [b + AQ^{-1} q]\\ &= -(Q^{-1} - Q^{-1} A^{\mathsf{T}} \bigl(AQ^{-1} A^{\mathsf{T}} \bigr)^{-1}AQ^{-1})q + Q^{-1} A^{\mathsf{T}} \bigl(AQ^{-1} A^{\mathsf{T}} \bigr)^{-1} b\\ &= -Rq + Cb. \end{aligned}\notag \end{equation}

The matrix \(\bigl(AQ^{-1} A^{\mathsf{T}} \bigr)^{-1}\) is, naturally, a Schur complement of the KKT matrix.

The actual value of the problem is got by substituting this expression back into the original objective function:

\begin{equation} \begin{aligned} J^\star(q, b) &= \frac{1}{2} q^{\mathsf{T}} R^{\mathsf{T}} Q Rq + \frac{1}{2}b^{\mathsf{T}} C^{\mathsf{T}} Q C b - b^{\mathsf{T}} C^{\mathsf{T}} Q R q - q^{\mathsf{T}} R q - q^{\mathsf{T}} C b\\ &=\frac{1}{2} \begin{bmatrix} q\\ b \end{bmatrix}^{\mathsf{T}} \begin{bmatrix} R^{\mathsf{T}} Q R - 2R & R^{\mathsf{T}} Q C\\ C^{\mathsf{T}} Q R & C^{\mathsf{T}} Q C \end{bmatrix} \begin{bmatrix} q\\ b \end{bmatrix} \end{aligned}\notag \end{equation}

Remark: The value of the problem will be a concave function of $q$. This is so because the original objective is a linear function of $q$, and the pointwise minimization of linear functions is concave. This is not obvious (to me?) from the cumbersome analytic expressions involved in expressing the value of the minimum, but can probably be shown directly by finding the right Schur complement.

The Markowitz Portfolio Problem

I generally try to avoid finance problems in this blog, but this example is incredibly famous, and a perfect illustration of quadratic optimization. Let us imagine that we have a collection of \(N\) risky assets available to invest in, and that we know the mean \(\mu \in \R^N\) and variance \(\Sigma \in \mathbb{S}_{++}^N\) of the (random!) future returns \(r \in \R^N\) of these assets. One particular version of the famous Markowitz portfolio problem is the following, for some “risk-aversion” parameter \(\lambda > 0\):

\begin{equation} \begin{aligned} \underset{x \in \R^n}{\text{minimize}}\ \frac{1}{2}\lambda x^{\mathsf{T}} \Sigma x^{\mathsf{T}} - \mu^{\mathsf{T}} x\\ \text{subject to}\ \mathbf{e}^{\mathsf{T}}x = w, \end{aligned} \tag{$\mathcal{M}_\lambda$} \end{equation}

where \(w\) is one’s initial capital. The constraint \(\mathbf{e}^{\mathsf{T}}x = w\) indicates that one is making a net investment of \(w\). It is worthwhile to mention that \(w\) can be zero (even negative) – the long side of the portfolio (i.e., everywhere with \(x_i > 0\)) can be financed by the short side of the portfolio (i.e., everywhere with \(x_i < 0\)).

This problem arises from the computation of the mean and variance of our actually realized returns from holding a portfolio \(x \in \R^{N}\). That is, our expected returns are \(\mathbb{E} r^{\mathsf{T}} x = \mu^{\mathsf{T}} x\) and the variance of those returns \(\text{Var}\ r^{\mathsf{T}} x = x^{\mathsf{T}} \Sigma x\). The objective function is reasonably well motivated by considering a quadratic approximation (or, leading to the same thing, assuming that \(r\) is Gaussian) of the exponential utility function \(\mathbb{E} \text{exp}(r^{\mathsf{T}} x)\), which is popular in decision theory .

I’ll give two derivations of the objective function in \(\mathcal{M}_\lambda\). We make two suppositions about an agent: firstly, they have an exponential utility function \(U( r) = 1 - e^{-\lambda r}\) over their total return. Secondly, the agent seeks to maximize their expected utility, or equivalently, to minimize \(\mathbb{E} e^{-\lambda r}\). The total return \(r\) is calculated as a weighted sum \(x^{\mathsf{T}} \mathbf{r} = \sum_{i = 1}^N x_i r_i\) where \(x \in \R^N\) is the agent’s decision variable, indicating how much capital to allocate to each security, and \(r_i\) is the (random) return of security \(i\). Thus, the agent is seeking to minimize \(J(x) = \text{exp}(-\lambda x^{\mathsf{T}} \mathbf{r})\).

Derivation 1 (Gaussian Returns): If we further suppose that the returns \(\mathbf{r}\) are normally distributed \(\mathbf{r} \sim \mathcal{N}(\mu, \Sigma)\) then \(J(x)\) is just the moment generating function for a multivariate Gaussian and admits the closed form \(\mathbb{E} \text{exp}\bigl(-\lambda x^{\mathsf{T}} \mathbf{r}\bigr) = \text{exp}\bigl(-\lambda \mu^{\mathsf{T}} x + \frac{1}{2}\lambda^2 x^{\mathsf{T}}\Sigma x\bigr)\). Then, since \(x \mapsto \mathsf{ln}(x)\) is a monotone increasing function, minimizing \(J(x)\) is equivalent to minimizing \(\mathsf{ln}\ J(x)\), which is (after removing a constant factor \(\lambda\)) exactly equivalent to the Markowitz objective.

Derivation 2 (Quadratic Approximation): The moment generating function \(\mathbb{E} \text{exp}(- \lambda x^{\mathsf{T}} \mathbf{r}\bigr)\) is not tractable in general for random variables \(\mathbf{r}\) – indeed, it is an integral in \(N\) dimensions. Instead of optimizing it directly, we can optimize a second order approximation as in \(e^z = 1 + z + \frac{1}{2}z^2 + \mathcal{O}(z^3)\). This leads to the approximation

\begin{equation} \begin{aligned} J(x) &\approx \mathbb{E}\bigl[ 1 - \lambda \mathbf{r}^{\mathsf{T} }x + \frac{1}{2}\lambda^2 (\mathbf{r}^{\mathsf{T} }x)^2 \bigr]\\ &= 1 - \lambda \mu^{\mathsf{T}} x + \frac{1}{2} \lambda^2 x^{\mathsf{T}} \mathbb{E}[\mathbf{r}\mathbf{r}^{\mathsf{T}}] x\\ &\equiv - \mu^{\mathsf{T}} x + \frac{1}{2} \lambda x^{\mathsf{T}} \Sigma x,\\ \end{aligned}\notag \end{equation}

where the \(\approx\) is up to third order in \(\mathbf{r}^{\mathsf{T}} x\), and \(\equiv\) is used to mean “equivalent for the purposes of optimization.

That the approximation is dropping moments of the returns higher than the second means that it does not account for skew (i.e., asymmetric return distributions), or kurtosis (i.e., heavy tails).

A primal-dual solution of this problem can be obtained by solving the corresponding KKT system

\begin{equation} \begin{bmatrix}\lambda\Sigma & \mathbf{e}\\ \mathbf{e}^{\mathsf{T}} & 0\end{bmatrix}\begin{bmatrix}x \\ \nu \end{bmatrix} = \begin{bmatrix} \mu \\ w \end{bmatrix},\notag \end{equation}

some straightforward JAX code implementing this follows. Using jax.vmap enables simultaneously computing an entire curve of solutions, and jax.jit dramatically speeds up this computation.

@jax.jit
@partial(jax.vmap, in_axes=[None, None, None, 0])
def solve_kkt(Sigma, mu, w, lmbda):
    """Solve the KKT system [[lmbda * Sigma, e]
                             [      e',      0]]"""
    n = len(mu)
    one = jnp.ones((n, 1))
    K = jnp.block([[lmbda * Sigma, one], [one.T, 0]])
    b = jnp.append(mu, w)
    z = jla.solve(K, b, assume_a="sym")
    return z[:-1], z[-1]

It is in fact possible to construct the entire solution curve for a wide range of parametric quadratic programs: the solution curve is piecewise linear. This has been known for probably 40 years and applies to, among other problems, LASSO regression and the SVM .

Example – The Markowitz Bullet

By way of example, I’ve used daily stock return data from Yahoo Finance to estimate (using the most elementary maximum-likelihood estimators) the (annualized) mean \(\mu\) and variance \(\Sigma\) of the log-returns \(r_t = \mathsf{ln}(1 + \frac{p_t}{p_{t - 1}})\) of 498 stocks (I presume drawn from the S&P500) over the last 1500 days. This is obviously a very long lookback window, but this serves only to get a some numbers which are quantitatively within a reasonable ballpark.

I’ve calculated various portfolios \(x \in \mathbb{R}^N\) along with their mean (log) returns \(\mu^{\mathsf{T}}x\) and standard deviations \(\sqrt{x^{\mathsf{T}} \Sigma x}\), which are plot in the following figures.

Breaking this down further, the dashed black line denotes the whole family of solutions \(x^\star(\lambda)\) to the Markowitz problem \(\mathcal{M}_\lambda\) (with \(w = 1\)). I’ve labeled it the “Pareto Front” because it is tracing the mean and standard deviation of Pareto Efficient portfolios – i.e., for any point on this curve, it is not possible to increase the mean of the returns without also increasing the standard deviation. However, for portfolios below the curve, like the individual stocks in the universe (denoted with squares), or the portfolios chosen at random (denoted with circles), which lie below this pareto curve, it is possible to increase the mean of the return without incurring any additional risk (as measured by the standard deviation). Moving from inside the pareto front towards the pareto front is a no-brainer! Points outside of the pareto curve are unreachable – there do not exist portfolios in this universe attaining that performance. This pareto front is referred to as the Markowitz Bullet.

Now, the Pareto front is not enough – we need to chose one portfolio of stocks. Surely the portfolio we choose should lie on the Pareto front, but where? One answer to this question is to choose the pareto efficient portfolio which also maximizes the Sharpe Ratio \(\mu / \sigma\), this point is indicated with a big red star – you can’t miss it.

There are a number of interesting observations to be made of these figures.

  1. None of the individual stocks lie on the efficient frontier – the benefits to diversification.
  2. The optimal Sharpe ratio of just under 2 when the universe is small (\(N = 10\)) is over \(5\times\) smaller than the optimal Sharpe ratio, over 11, with a larger universe (\(N = 498\)). It’s better to have more options.
  3. The random portfolios, even though they are scaled to the max SR portfolio, cover much less space in high dimensions than in low dimensions. Indeed, in high dimensions, the optimal point is in some sense “more exceptional” than in low dimensions.

Remark (Statistical Arbitrage): Interestingly, it is not necessarily the case that $\mu_i < 0 \implies x_i^\star \le 0$. That is, it can be optimal to make a positive investment (go long) in a stock with strictly negative returns. The reason is that this stock can serve as an hedge. For example, if it is negatively correlated with another stock having positive returns, the combination of these two may still have positive returns, but lower overall variance than either one alone. Example – Perturbations

In the previous example, I’ve calculated optimal portfolios (by solving \(\mathcal{M}_\lambda\)), and evaluated them (calculating their mean and standard deviation), using the same statistics \(\mu, \Sigma\). That is, we have assumed that the second order statistics of returns are exactly known. This is highly unrealistic. Indeed, it is quite pointless to evaluate or simulate the performance of some algorithm (whether in finance or science in general) under the exact same conditions you optimized for. This tells you nothing.

Indeed, evaluation should always be more difficult and more realistic than is the optimization problem that you actually solve. The optimization problem is merely a tractable approximation to reality.

A Margin of Safety

One part of why \(x^\star_{\mathcal{M}}\) can make for a poor choice is that the estimates \(\mu\) of forward returns are extremely uncertain. Rather than relying on \(\mu\) directly, we should practice some epistemic humility and ensure that this is some margin of safety \(M\) in our estimate of \(\mu\). That is, we allow that there is some error term \(\zeta\) satisfying a norm bound \(||\zeta|| \le M\) and that the actual mean forward returns are given by \(\mu + \zeta\). Then, in order to construct a robust and conservative portfolio, consider the minimax portfolio problem

\begin{equation} \begin{aligned} \underset{x \in \R^n}{\text{minimize}}&\ \underset{\zeta: ||\zeta|| \le M}{\text{max}}\ \frac{1}{2}\lambda x^{\mathsf{T}} \Sigma x^{\mathsf{T}} - (\mu + \zeta)^{\mathsf{T}}x\\ \text{subject to}&\ \one^{\mathsf{T}} x = w. \end{aligned} \tag{$\mathcal{C}$} \end{equation}

That a solution \(x^\star_{\mathcal{C}}\) (\(\mathcal{C}\) for “conservative”) actually exists follows from the celebrated minimax theorem . Indeed, the problem \((\mathcal{C})\) can be seen as a type of game wherein the “\(x\) player” is trying to minimize, and the “\(\zeta\) player” is trying to maximize. If we imagine ourselves as the $x$-player trying to optimize our investment return, the \(\zeta\) player is an adversary keeping us honest.

One of the remarkable consequences of the minimax theorem is that the order of “min” and “max” can be swapped, which is not at all true in general . Indeed, it is a fact that the following optimization problem is exactly equivalent to \((\mathcal{C})\)

\begin{equation} \underset{\zeta: ||\zeta|| \le M}{\text{maximize}}\ \underset{x \in \R^n: \one^\T x = w}{\text{min}}&\ \frac{1}{2}\lambda x^{\mathsf{T}} \Sigma x^{\mathsf{T}} - (\mu + \zeta)^{\mathsf{T}}x, \tag{$\mathcal{C}^\prime$}\\ \end{equation}

and using our knowledge of quadratic forms, we know that the inner minimization problem is a linear function of \((\mu + \zeta)\) and \(w\), as in \(x^\star = R(\mu + \zeta) + cw\), where \(c \in \R^n\) may be thought of as a “biasing vector”. The exact form of the matrix \(R\) has been given in an earlier section.

As well, using the fact that the minimum over \(x\) will be a convex function in \(\zeta\) (mentioned earlier) the cost function can be written, up to constant factors, as another quadratic form in \(\zeta\) (using generic \(Z \succ 0\) and \(z\)):

\begin{equation} \underset{\zeta: ||\zeta|| \le M}{\text{max}}\ -\zeta^{\mathsf{T}} Z \zeta + z^{\mathsf{T}} \zeta. \tag{$\mathcal{C}^{\prime\prime}$} \end{equation}

Given an optimal \(\zeta^\star\) for this problem, a solution to the original problem can be obtained through the substitution \(x^\star_{\mathcal{C}}(\lambda) = x^\star_{\mathcal{M}} (\mu + \zeta^\star; \lambda, w)\), since \(x^\star_{\mathcal{M}}(\mu + \zeta^\star; \lambda, w) = R(\mu + \zeta) + cw\) is the solution of the inner minimization.

It is now a question of whether or not we can actually solve \(\mathcal{C}^{\prime\prime}\). The reader with experience in convex optimization will know that the answer is a clear affirmative. Indeed, the objective function is concave in the variable \(\zeta\) over which we are maximizing, and the constraint set \(\zeta: ||\zeta|| \le M\) is convex.

In fact, this problem is an instance of a trust region subproblem (TRSP), which arises in iterative optimization algorithms. More specifically, a trust region algorithm approximates the (non-quadratic) objective function by a quadratic form which is valid within some bounded region. The quadratic problem is solved in this region to obtain a new candidate point, wherein the procedure continues with a new quadratic approximation centered upon this point. The algorithm continues until convergence.

The problem \(\mathcal{C}^{\prime\prime}\) can be solved computationally for more-or-less any norm on \(\zeta\). The infinity norm \(||\zeta||_\infty\), corresponds exactly to the margin of safety, but the quadratic norm \(||\zeta||_2^2 \le M^2\) admits of an elegant analytic solution relating back to eigenvalue problems.

The Quadratic TRSP

Let us specialize problem \((\mathcal{C}^{\prime\prime})\) to the case of a quadratic norm – this is the usual instance referred to as “the” TRSP:

\begin{equation} \underset{\zeta: ||\zeta||_2^2 \le M}{\text{min}}\ \zeta^{\mathsf{T}} Z \zeta - z^{\mathsf{T}}\zeta \notag \end{equation}

where I’ve also reversed the sign in the objective in order to treat it as a minimization problem, the conventions of which I am more used to. We can start to approach this problem by manipulating a Lagrangian function with dual variable \(y \ge 0\):

\begin{equation} \begin{aligned} \mathcal{L}(\zeta, y) &= \zeta^{\mathsf{T}} Z \zeta - z^{\mathsf{T}} \zeta + y(||\zeta||_2^2 - M)\\ &= \zeta^{\mathsf{T}} (Z + yI) \zeta - z^{\mathsf{T}}\zeta - yM \end{aligned}\notag \end{equation}

Minimizing this quadratic form over the variable \(\zeta\), which has optimal value \(\zeta^\star(y) = -(Z + yI)^{-1} z\), leads to a scalar dual function \(g: \R \rightarrow \R\)

\begin{equation} \begin{aligned} g(y) &= \underset{\zeta \in \R^N}{\text{inf}}\ \mathcal{L}(\zeta, y)\\ &= -z^{\mathsf{T}} (Z^{} + yI)^{-1} z - yM. \end{aligned}\notag \end{equation}

The optimal value of \(\zeta\) is obtained by optimizing this scalar function \(g(y)\) and substituting the optimal \(y^\star\) back into the function \(\zeta^\star(y^\star)\).

We can make further headway on this problem by using the eigenvalue decomposition of \(Z\). Indeed, since \(Z \succ 0\), it admits of an eigenvalue factorization with strictly positive eigenvalues \(\gamma_1, \ldots, \gamma_N\) and an orthogonal basis \(V\): \[Z = V\Gamma V^{\mathsf{T}} = \sum_{i = 1}^N \gamma_i v_i v_i^{\mathsf{T}.}\] Thus, we can write

\begin{equation} \begin{aligned} g(y) &= - z^{\mathsf{T}} (Z + yI)^{-1} z - yM\\ &= - z^{\mathsf{T}} \bigl(V(\Gamma^{} + yI)V^{\mathsf{T}} \big)^{-1} z - yM\\ &= - z^{\mathsf{T}} V(\Gamma + yI)^{-1} V^{\mathsf{T}} z - yM\\ &= - z^{\mathsf{T}} \bigl(\sum_{i = 1}^N \frac{1}{\gamma_i + y} v_i v_i^{\mathsf{T}} \bigr) z - yM\\ &= - \sum_{i = 1}^N \frac{1}{\gamma_i + y} (v_i^{\mathsf{T}} z)^2 - yM.\\ \end{aligned}\notag \end{equation}

Since \(\gamma_i > 0\) for each \(i\), and \(y \ge 0\) by the duality constraint, we are in no danger of dividing by zero in this summation (this puts us in the easy version of the TRSP ).

To obtain an optimal dual variable \(y^\star\) we need to maximize \(g\) over \(y \ge 0\). It may not be particularly easy to recognize from this expression (although it is necessarily true as a general theorem) but the function \(g\) is a concave function of \(y\) on this interval since the function \(x \mapsto 1/x\) is convex for \(x \ge 0\), and composition of convex functions with linear functions remain convex.

Since it is clear that \(g(y) \rightarrow -\infty\) as \(y \rightarrow \infty\), there must exist a solution. Thus, we can find the optimal \(y\) by differentiating \(g\) and then carrying out bisection on the derivative. Explicitly,

\begin{equation} g^{\prime}(y) = \sum_{i = 1}^N \frac{(v_i^{\mathsf{T}} z)^2}{(\gamma_i + y)^2} - M.\notag \end{equation}

The optimal dual multiplier will satisfy \(y^\star > 0\) only if \(g^{\prime}(0) > 0\). This is so since we seek to find a zero \(g(y^\star) = 0\) and since \(g^{\prime}\) is monotone decreasing such a zero does not exist unless \(g^{\prime}(0) > 0\).

In the case where \(g^{\prime}(0) \le 0\) we must necessarily have \(y^\star = 0\) and thus \(\zeta^\star(0) = -Z^{-1} z\). In fact, if \(w = 0\), this reduces to \(x^\star = 0\), i.e., invest nothing. In such a case, none of the expected returns are large enough to exceed our margin of safety. Indeed, in the \(w = 0\) case, the equations reduce into

\begin{equation} \begin{aligned} g^\prime(0) &= \frac{1}{2\lambda} \sum_{i = 1}^N (v_i^{\mathsf{T}} \mu)^2 - M\\ &= \frac{1}{2\lambda} \mu^{\mathsf{T}} VV^{\mathsf{T}} \mu - M\\ &= \frac{1}{2\lambda} ||\mu||_2^2 - M, \end{aligned}\notag \end{equation}

so we can explicitly recognize that our optimal investment would be zero exactly when \(\frac{1}{2}||\mu||_2^2 \le \lambda M\). That is, the “energy” (squared norm) of our nominal returns need to exceed the margin of safety scaled by our risk aversion. When \(w \ne 0\), we still need to be fully invested, but the optimal solution will be the one which is “maximally robust”.
